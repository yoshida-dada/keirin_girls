# S2 特徴量エンジニアリング＋学習 設計（C）

対象: 仕様書 `girls_keirin_ai_spec.md` 2章（特徴量）＋Phase2後半（確率モデル）＋課題B/E（キャリブレーション/リーク）。
前提: S1データ収集・S3ベースライン（競走得点→PL→レースタイプ）・S4 EVエンジンは実装済み。

---

## 0. 設計の最重要方針：リーク防止（課題E / 2.7）

学習・バックテストは**過去レース**に対して行う。ここで最大の落とし穴は
「選手成績を『今日時点』の値で引くと未来情報が漏れる」こと。例えば競輪ステーションの
「直近4ヶ月」集計は**現在のスナップショット**で、2025年のレースに使うと答えを見てしまう。

**→ 特徴量は必ず「そのレース発走前に確定していた値（as-of）」だけで作る。** 具体的な出所を2系統に限定する:

1. **レースページ同梱の as-of スタッツ（第一次ソース・リークなし）**
   GambooBET のオッズ/出走表ページは、そのレース時点の
   `競走得点`／`直近4ヶ月成績`（勝率・2/3連対率・脚質別回数=逃捲差マ・着順分布・S/B/H回数）を
   **当時の値のまま**埋め込んでいる。過去レースのページを取れば as-of スタッツがそのまま得られる。
   → **まず racecard パーサを拡張してこの直近4ヶ月ブロックを抽出する**（S2の最初のタスク）。

2. **自前の収集済み結果からの as-of ローリング（第二次ソース・リークなし）**
   `results` テーブルに貯めた各選手の過去着順・決まり手・上がりタイムから、
   「そのレース日より前」だけを使って直近N走・期間勝率・脚質傾向を再計算する。
   バンク別成績もここから（会場でフィルタ）。長期の地力・成長率もここで作れる。

**競輪ステーションの現在スナップショット（keirin_station.py）は「当日予測（ライブ）」専用**とし、
過去の学習には使わない。役割は (a) rider_id 解決、(b) ライブ予測時の直近成績、(c) 長期career補完。

## 2. 特徴量カタログ（2章 → 出所 → 式 → リーク）

### 2.1 選手単位（Tier1）
| 特徴量 | 式 / 出所 | リーク対策 |
|---|---|---|
| racing_score | racecard 競走得点 | as-of |
| win_rate / top2_rate / top3_rate | racecard直近4ヶ月 or results as-ofローリング | as-of |
| recent_finish_5 / _10（直近着順平均） | results as-of（直近N走） | 過去のみ |
| leg_type（逃/捲/差/マ）＋各率 | racecard脚質＋直近4ヶ月決まり手率 | as-of |
| bank_win_rate（当該会場） | results as-of（会場フィルタ）※初期は薄い | 過去のみ |
| bracket_number（枠番） | entries | as-of |
| rel_score_mean = racing_score − レース平均 | entries集計 | as-of |
| rel_score_max = racing_score − レース最高 | entries集計 | as-of |
| score_rank_in_race（レース内得点順位） | entries集計 | as-of |

### 2.2 レース全体（2.5 / 混戦度）
得点標準偏差・トップとの得点差・レース平均得点・脚質構成（逃/差人数比）・年齢分布。
→ 全て entries から算出。**混戦度指標（race_type）と重複**するので S3 の入力にも供給。

### 2.3 派生（2.8）
| 特徴量 | 式 |
|---|---|
| 好調指数 | 直近4ヶ月着順平均 − career着順平均（低いほど好調） |
| 安定指数 | 着順の標準偏差（results as-of） |
| 逃げ期待値 | 本人逃げ率 × レース内逃げ型人数 |
| 展開有利指数 | 本人差し率 × レース内逃げ型人数 |
| バンク適性指数 | 当該会場勝率 − 全体勝率（データ蓄積後） |
| 成長率 | 競走得点の推移（racecard/resultsの時系列差分。初期は薄い） |

Tier2/3（初手位置・打鐘位置・オッズ変動・コメントNLP）はデータ蓄積後。

## 3. モジュール構成（`src/features/`）

```
src/features/
├── rider_features.py    # 選手単位: racecard直近4ヶ月抽出 + results as-ofローリング
├── race_features.py     # レース全体: 得点分散/平均/脚質構成/年齢分布/相対得点/順位
├── derived_features.py  # 派生: 好調/安定/逃げ期待値/展開有利/バンク適性/成長率
└── assembler.py         # 1レース → 特徴量行列 (car_number × feature) を組む。as-of日付を厳守
```

- 入力は DataFrame（`pd.DataFrame(entries)` 等）。関数は純関数でテスト可能に。
- `assembler.build(race_id, as_of_date)` が「そのレースの発走前情報のみ」で特徴量を組む。
- results as-of は `WHERE race_date < :as_of` を必須にしてリークを機械的に防ぐ。

## 4. 学習（Phase2後半 / S3の強化）

### 4.1 現状（ベースライン・実装済み）
`strength.py`: 強さ s_i ∝ exp((競走得点−平均)/温度) → PL で210通り。競走得点のみ。

### 4.2 目標（学習モデル）
特徴量ベクトル x_i から強さ score_i を学習する2案:
- **A: Plackett-Luce の MLE/勾配学習** — 特徴量の線形（or 浅いNN）結合で strength を出し、
  実着順の順序尤度を最大化。全210通り確率が自然に出る（仕様書の本命）。
- **B: LightGBM lambdarank** — レース内ランキング学習。強力だが確率化に別途キャリブレーションが必要。

**決定（2026-07-14）: A（PL線形）と B（LightGBM lambdarank）を両方併走**し、Phase4のバックテストで
ROI と Brier を比較して優位な方を採用する。確率整合の取りやすいAを主、非線形補足としてBを従で持つ。

### 4.3 キャリブレーション（課題B・必須）
- 学習後、reliability curve と Brier score を **オッズ帯 × レースタイプのバケット別**に検証。
- ここが崩れているバケットは Phase4 で運用除外。→ `src/backtest/` と共通の検証器を作る。

### 4.4 学習データの作り方
- 各過去レースを `assembler.build(race_id, as_of=race_date)` で特徴量化。
- ラベル = results の着順（PLは順序、GBDTはrank）。
- 時系列分割（過去で学習→未来で検証）でリークなく評価。

## 5. 実装順（S2）

1. **racecard 直近4ヶ月ブロックのパース拡張**（as-of スタッツの第一次ソース確保）。
2. `race_features.py`（相対得点・得点分散・脚質構成・混戦度）。entriesだけで完結、即着手可。
3. `rider_features.py`（racecard直近4ヶ月 → 特徴量化。results as-ofローリングは蓄積待ち）。
4. `derived_features.py`（逃げ期待値・展開有利・好調/安定指数）。
5. `assembler.py`（as-of厳守で1レース→特徴量行列）。
6. 学習器 `src/model/train_pl.py`（PL特徴量線形）＋時系列CV＋Brier/reliability。
7. `strength.py` を学習済み係数で置換（競走得点単体→特徴量ベース）。

## 6. 決定事項（2026-07-14）
- 学習モデル: **A（PL線形）＋ B（LightGBM lambdarank）を両方併走**し、バックテストのROI/Brierで選ぶ。
- バンク別・長期career: **自前resultsの蓄積を待ち、as-of再計算で作る**（リーク完全回避）。
  keirin_stationのcareer現在値は過去学習には使わない（ライブ予測のみ）。データが薄い初期は
  当該特徴量を欠損許容（モデル側でNaN対応）とし、蓄積に伴って効かせる。
- 直近N走・期間: 初期値 N=5/10、期間=30/90日で実装し、SHAP/重要度で見直す。

## 7. 残る小さめの確認（実装時に判断）
- PL線形の最適化（scipy.optimize で順序尤度最大化 or 勾配）。少数レースでも安定する正則化。
- 特徴量欠損（初期のバンク別など）の埋め方（レース平均/中央値 or NaNのままGBDTに委ねる）。
