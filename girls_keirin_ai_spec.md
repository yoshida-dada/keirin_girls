# ガールズケイリン 期待値予測AI 開発仕様 v3（課題整理＋特徴量仕様＋タスク一覧）

## 0. コンセプト

**全レース・全210通りの3連単確率を推定し、「レースタイプ × オッズ帯」ごとに期待値の出るゾーンを特定して買い目を選定する。**

1着固定などの制約は課さない。Plackett-Luce型モデルなら全買い目の確率が自然に出るため、構成は以下の3段。

1. **確率モデル**：出走メンバーから3連単210通り全ての的中確率を推定
2. **レースタイプ分類**：確率分布の形状から「軸堅／標準／混戦」を定量判定
3. **EVゾーン特定**：過去データのバケット分析で「どのタイプ × どのオッズ帯」に実現期待値があるかをマッピングし、そのゾーン内でEV閾値を超える買い目のみ購入

## 1. 主要課題

### 課題A：EVの源泉の特定（最重要）
3連単の控除率は25%。期待値が出るのは「モデル確率 > オッズ逆算確率」となる歪みが33%以上あるゾーンだけ。
→ 過去レースを（レースタイプ × オッズ帯）でバケット分割し、各バケットで「EV閾値超えの買い目を機械的に全買いした場合の実現ROI」を集計。ROI 100%超のバケットだけを実運用の対象にする。
→ favorite-longshot bias（穴側が過剰に売れる傾向）がガールズ3連単プールでも成立するかをデータで確認する。

### 課題B：バケット別キャリブレーション
混戦レースは1点あたり確率1〜3%が主戦場になり、モデル誤差がEV判定を直撃する。
→ オッズ帯・レースタイプごとにreliability curve・Brier scoreで検証。ROIが良く見えてもキャリブレーションが崩れているバケットは運用から除外する。

### 課題C：混戦度（レースタイプ）の定量化
「軸が堅いか混戦か」を再現可能な指標に落とす。候補：トップ選手の1着確率、上位2名と3位以下の得点差、1着確率分布のエントロピー。
→ どの指標・閾値が「EVゾーンの分離」に効くかはPhase 4で検証。

### 課題D：締切直前のオッズ下落
穴目は締切直前に大きく下がる。「表示20倍で買ったら確定14倍」が常態化するとEVが消える。
→ オッズ時系列スナップショットを蓄積し、オッズ帯別のhaircut係数を推定（Baneiのodds haircut実装を流用）。バケットROI集計もhaircut後オッズで行う。

### 課題E：ガールズ特有の構造（ラインなし）
ラインがないため、「誰と走るか」より「各選手の能力・状態・展開適性」が結果に直結する。単純な競走得点だけでは不十分で、当日の状態・レース内での相対関係・展開を表現する特徴量設計が精度を左右する（詳細は2章）。
→ 7車立て固定で組み合わせ空間は210通りと小さいが、位置取り・脚質構成の影響は大きく残るため特徴量側でカバーする。
→ 選手数が少なく開催も限られる → データ蓄積に時間。バケットを細かくしすぎるとサンプル不足で結論が出ない点に注意。

### 課題F：戦略の分散と資金管理
混戦×高オッズ狙いは的中頻度が低く、ROI収束に数百〜数千レース必要。
→ quarter Kellyでの投票額管理と累積ROIの永続記録を最初から組み込む。バケット別に成績を分けて記録し、機能していないゾーンを随時停止できる設計にする。

### 課題G：データ取得
オッズ時系列（下落率推定＋バケット分析用）に加え、特徴量用の選手成績データ（バンク別・脚質・直近成績等）の定期取得が必要。取得間隔・規約・Raspberry Pi上での安定稼働を設計する。

## 2. 特徴量仕様

ラインがない競技特性上、「絶対能力」だけでなく「レース内での相対関係」と「展開」を表現できる特徴量が精度を左右する。7カテゴリに分けて設計する。

### 2.1 選手の基礎能力（土台）
競走得点、勝率、連対率、3連対率、平均着順、級班。
→ 競走得点が近い選手でも「1着5回・7着5回」と「3着10回」では性質が異なるため、勝率・連対率・3連対率を併用して安定感と勝ち切る力を分離する。

### 2.2 直近コンディション（重要）
直近5走・10走着順、直近30日／90日勝率、中何日、開催連戦日数、前日落車・失格歴・欠場明け。
→ 競走得点は更新が遅く「現在の調子」を反映しにくいため、直近成績で補完する（年間勝率40%でも直近5走が7・6・6・5・7なら不調、という判定を可能にする）。

### 2.3 バンク適性
バンク別勝率・3連対率、当該競輪場成績、バンク周長（333m/400m/500m）、屋内外、ナイター／ミッドナイト適性。
→ 競馬の芝巧者・ダート巧者に相当。全体勝率25%でも特定バンクだけ勝率45%といった選手差を捉える。

### 2.4 レース展開特徴（ガールズでは特に重要）
脚質（逃・捲・差・マーク）、逃げ率・捲り率・差し率・マーク率、初手位置予測、前受率、打鐘位置・最終ホーム／バック位置。
→ 同じ能力でもメンバー構成で勝率が変わる（逃げ選手が3人いれば前が潰れやすく、1人なら楽逃げになりやすい）ため、個人特徴量だけでは不十分。

### 2.5 レース全体特徴（見落とされやすいが重要）
得点分散（標準偏差）、トップとの得点差、レース平均得点（クラス・開催格の代理）、脚質構成（逃げ／差しの人数比）、年齢分布。
→ 混戦度分類（課題C）に直結する。得点が僅差で並べば混戦、突出した1位がいれば軸堅、と判定できる。

### 2.6 相対能力特徴量（推奨・効果大）
本人得点−レース平均、本人得点−レース最高点、レース内得点順位。
→ 「絶対能力」より「このレース内で何番目か」をモデルが学習しやすくなり、競走得点単体より効くことがある。

### 2.7 市場情報（学習利用は慎重に）
前日オッズ、当日朝オッズ、発売開始オッズ、人気順位。
→ 最終オッズは期待値計算（EVエンジン側）に使うため、学習特徴量として使う場合はリーク防止の設計が必要（例：確定直前オッズを学習に使うと市場の答えを写すだけになりEV源泉が消える）。

### 2.8 派生特徴量（Feature Engineering・効果大）
能力差（本人得点−平均得点）、最近好調指数（直近5走平均着順−年間平均着順）、安定指数（着順標準偏差）、逃げ期待値（逃げ率×逃げ人数）、展開有利指数（差し率×逃げ人数）、バンク適性指数（当該バンク勝率−全体勝率）、成長率（90日前競走得点→現在競走得点の変化）。
→ 人間の競輪予想で暗黙的に考えている要素を数値化したもので、モデルがパターンを学習しやすくなる。

### 2.9 採用優先度

**Tier1（必須・初期モデルから搭載）**
競走得点／勝率・連対率・3連対率／直近5走・10走成績／脚質（逃・捲・差・マーク）／バンク成績／枠番／相対得点（平均差・最高点差）／レース内得点順位

**Tier2（高効果・データ整備でき次第追加）**
初手位置予測／中何日／落車・失格・欠場明け／バンク種別（333m/400m/500m）／ナイター・ミッドナイト適性／得点分散／脚質構成／混戦指数（エントロピー等）

**Tier3（データ蓄積後に追加検討）**
打鐘位置・最終バック位置などの位置取り指標／オッズ変動パターン／選手コメントの自然言語解析（LLM）／過去対戦相性／季節・気温・風向きなどの環境要因

### 2.10 設計原則：特徴量はレース単位で設計する
「選手Aが強い」だけでなく「選手Aがこの6人と走るときにどれだけ有利か」が重要。選手ごとの特徴量に加え、レース平均得点・得点標準偏差・上位2名と残り5名の能力差・逃げ型/差し型の人数構成・混戦指数といった**レース全体を表現する特徴量**をセットで設計する。出走人数が少なく相対関係が結果に強く影響するガールズケイリンでは、この設計がPlackett-Luce等の順位モデルや勾配ブースティング系モデルの性能を大きく左右する可能性がある。

## 4. 最初の一歩（推奨順）

1. Phase 1のスキーマ設計＋過去データ取得（確定オッズは**全点**取得が必須。選手成績詳細も同時に集める）
2. Tier1特徴量＋Plackett-Luceベースラインで210通り確率＋混戦度指標を1レース分出力してみる
3. オッズスナップショット収集を即開始（haircut推定はデータが貯まるまで時間がかかるため先行着手）
4. データが揃い次第、Phase 4のバケットROI集計へ。**狙い目マップができるまでは実弾投入しない**
